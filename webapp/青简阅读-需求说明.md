# 青简阅读 · 开发目标与需求说明
> 基于成熟阅读平台调研梳理（微信读书 / 番茄小说 / 起点读书 / 掌阅 iReader / Kindle / QQ阅读 / 多看阅读 / legado 开源阅读）
> 文档用途：作为本项目（书源聚合网页阅读器）的后期开发目标与功能基线，供"逐项对照、查漏补缺"使用。

---

## 一、调研对象与定位

| 平台 | 核心定位 | 资源形态 | 阅读器特点 |
|------|----------|----------|------------|
| 微信读书 | 社交化读书（微信关系链） | 正版出版书+网文+有声 | 界面简洁、排版清爽、可划线批注、看好友动态 |
| 番茄小说 | 免费网文（字节系） | 海量正版网文，免费+广告 | 阅读门槛低、AI 朗读、夜间模式顺滑 |
| 起点读书 | 网文追更 | 原创网络文学、更新快 | 适合追更、章节进度清晰 |
| 掌阅 iReader | 综合老牌 | 出版书+网文，排版近纸质 | 自定义字体/背景/翻页动画丰富 |
| Kindle | 专业深度阅读 | 全球电子书、多格式 | 阅读体验专业、云同步 |
| QQ阅读 | 网文+听书 | 腾讯系网文/畅销书 | 更新快、适合追更 |
| 多看阅读 | 学习资料 | 多格式、PDF 支持好 | 界面干净、功能实用 |
| **legado 开源阅读** | **书源聚合工具** | **不自带资源，靠导入书源** | **多书源同时搜、一键换源、全自定义排版、无广告开源** |

**关键结论**：前 7 个都是"自带书城的封闭平台"，只有 **legado** 与本项目同构——**本身是工具，资源靠书源聚合**。因此青简阅读的对标主体应是 **legado（功能内核）+ 微信读书/掌阅（体验外壳）**。

## 二、成熟平台的共性（提炼）

无论平台定位如何，以下能力是"所有成熟阅读产品都具备"的基线，可作为本项目的最低功能门槛：

### 2.1 三大核心模块（无一例外）
1. **发现/书架**：进入即见"在读书 + 想读书"，一键续读。
2. **阅读器**：沉浸式全屏，所有阅读控制集中可唤出。
3. **个人/设置**：阅读偏好持久化、阅读数据、账号/关于。

### 2.2 阅读器共性功能（必做项，按出现频率排序）
- ✅ **翻页模式**：滚动 / 翻页（仿真、滑动、覆盖）多选项 —— 100% 具备
- ✅ **字号 + 字体 + 行距 + 段距 + 加粗** 自定义 —— 100%
- ✅ **背景主题**：纸白 / 护眼绿 / 夜间深色 / 羊皮纸 —— 100%
- ✅ **亮度调节**（独立遮罩，不影响系统） —— 95%
- ✅ **目录抽屉**（左侧，章节可折叠、当前章高亮、点选跳转） —— 100%
- ✅ **进度可视化**：底部细进度条 + 章节内页码（如"第 3 / 8 页"或"73%"） —— 100%
- ✅ **上一章 / 下一章** 快速切换 —— 100%
- ✅ **书签 / 进度记忆**（自动保存上次位置，下次续读） —— 100%
- ✅ **护眼 / 夜间模式** 一键切换 —— 100%
- 🔶 **自动翻页**（定时翻页，通勤场景） —— 70%
- 🔶 **听书 / TTS 朗读** —— 70%（网文平台标配）
- 🔶 **笔记 / 划线 / 批注** —— 60%（出版书平台侧重）
- 🔶 **阅读统计**（时长、完成率、打卡） —— 50%

### 2.3 首页 / 书架共性
- 书架支持 **网格 / 列表** 两种视图切换（legado、掌阅均支持）。
- "继续阅读"横幅置顶，点封面直接进正文（本项目已实现"书架直进正文"，领先多数）。
- 排序维度：最近阅读 / 加入时间 / 书名（石头阅读、掌阅均支持）。
- 发现页：热门榜单、新书速递、编辑精选（封闭平台有，书源聚合型可后置）。

## 三、开发目标（青简阅读定位）

**一句话目标**：做一个**免费、开源、无广告**的网页版 legado —— 用书源聚合搜全网小说，提供"接近微信读书"的沉浸式阅读体验。

**设计原则（沿用已确立的视觉规范）**：
- 配色：浅灰底 + 黑白灰 + 蓝绿点缀（`--accent:#12b3a6`），**不用起点红**。
- 布局：扁平、大留白、卡片式封面、**底部标签导航**（搜索 / 书架 / 我的）。
- 阅读器：全屏沉浸，点击正文中间唤出控件，参考微信读书 / 番茄 / Kindle。

**技术边界（用户已明确）**：
- 书源本身大量失效/反爬，无法"全打通"；重点是**用户端体验扎实**（可用源优先、失败优雅兜底）。
- 自测靠"审查→想象→改→循环"，不装无头浏览器、不远程操控。

## 五、功能优先级总览

- **P0（必须有，对齐成熟平台底线）**：底部导航、书架直进续读、搜索合并多源、换源、沉浸式阅读器（滚动/翻页+上一张下一张+翻页动画+页码）、字号/行距/段距/加粗/字体/背景主题/亮度、目录抽屉、进度条、书源记忆+失效剔除、gbk 不乱码、失败兜底。
- **P1（应做，体验完整性）**：书架网格/列表切换、排序、书签、护眼一键、书籍元数据、设置总开关、空状态文案强化。
- **P2（可选，差异化）**：自动翻页、TTS 听书、阅读统计/打卡、发现/榜单、高级筛选、笔记划线、数据导出、关于页。

## 七、给开发的执行建议（循环自测法）

按用户既定方式推进：每补一项功能 → 静态审查（作用域/幽灵引用/事件）→ 想象真实用户路径 → 改 → 跑 `_tests/pre_delivery_check.js` 直到 175/175 → 再循环。绝不装无头浏览器、绝不远程操控；改动后提醒用户 **Ctrl+Shift+R 硬刷新**、若动 `server.js` 需 **重启 node server.js**。
